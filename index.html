Operational Notes · Reliability · Distributed Systems

I build distributed systems that behave under stress. These are short notes about reliability, incident response, and the human coordination layer that determines whether a system is actually stable.

Principle

Make failure predictable

SLOs, error budgets, and runbooks exist to remove surprise—not to create dashboards.

Principle

Recovery should be boring

If recovery depends on heroics, the system is already in debt.

Principle

Artifacts > vibes

Write it down. Ship something. If it doesn’t produce an artifact, it won’t survive load.

Principle

Coordination is a failure domain

Most outages are coordination failures wearing a latency mask.
